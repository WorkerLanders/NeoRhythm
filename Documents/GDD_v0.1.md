# NeoRhythm — 게임 기획서 초안 (GDD v0.1)

## 개요

Windows 플랫폼 대상의 횡스크롤 리듬 게임. 노트가 우→좌로 이동하며 판정선 도달 시 마우스 클릭 또는 키 입력으로 타격하는 방식. 유저와 가족의 실제 인생 경험을 바탕으로 한 스토리 모드를 핵심으로 하며, AI 음악 생성 서비스를 통해 이야기에 맞는 음악을 제작한다.

---

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| 장르 | 횡스크롤 리듬 액션 |
| 플랫폼 | Windows (PC) |
| 입력 방식 | 마우스 좌클릭 (단일 입력) |
| 시점 | 2D 사이드뷰 |
| 타겟 유저 | 리듬 게임 입문~중급자 |

---

## 2. 핵심 메카닉

### 2-1. 레인 구성
- 화면을 수평으로 **N개 레인**으로 분할 (기본 4레인, 난이도별 증가 고려)
- **모든 레인은 단 하나의 입력(마우스 좌클릭)에 대응** — 레인 구분은 시각적 다양성 목적
- 판정선은 화면 **좌측 약 15% 지점**에 수직으로 고정
- **동시에 2개 이상의 레인에 노트가 나오는 경우 없음** (채보 설계 원칙)

### 2-2. 입력 방식
- **마우스 좌클릭 고정** (단일 입력)
- 어느 레인의 노트든 같은 클릭 하나로 처리
- 클릭 타이밍만이 판정에 영향을 미침

### 2-3. 노트 종류 (기본 3종)
| 종류 | 설명 | 입력 |
|------|------|------|
| **탭 노트** | 기본 단타 노트 | 마우스 좌클릭 단타 |
| **홀드 노트** | 길게 누르는 노트 (롱노트) | 마우스 버튼 누름 유지 |
| **슬라이드 노트** | 연속 연타 (드럼롤 느낌) | 연속 클릭 |

### 2-4. 노트 흐름
- 노트 생성 위치: 화면 오른쪽 바깥 (오프스크린)
- 이동 방향: 우 → 좌 (일정 속도, 노래 BPM 기반으로 스크롤 속도 조절 가능)
- 한 시점에 활성 노트는 항상 **1개 레인**에만 존재
- 판정선 통과 시 판정 발생 → 통과하면 Miss 처리

---

## 3. 판정 시스템

| 판정 | 타이밍 오차 | 배율 |
|------|------------|------|
| **PERFECT** | ±20ms | ×1.0 |
| **GREAT** | ±50ms | ×0.8 |
| **GOOD** | ±100ms | ×0.5 |
| **MISS** | 초과 / 미입력 | ×0 (콤보 리셋) |

- 판정 결과는 판정선 위에 텍스트 + 이펙트로 즉시 표시
- 화면 하단에 **판정 히트바** (오차 분포 시각화) 표시 고려

---

## 4. 스코어링 & 콤보

- **기본 점수**: 노트당 1000점 × 판정 배율
- **콤보 보너스**: 콤보 수에 따라 추가 배율 (50콤보: ×1.1 / 100콤보: ×1.2 / 200콤보: ×1.5 등)
- **MAX 콤보**: 곡 내 모든 노트 PERFECT 시 달성
- **등급 시스템**: S+ / S / A / B / C / D (정확도 % 기반)

---

## 5. 체력(HP) 시스템 (선택적)

- HP 바를 화면 일측에 표시
- MISS 시 HP 감소, PERFECT/GREAT 시 소폭 회복
- HP 0 → 곡 실패 (arcade 모드)
- 연습 모드에서는 HP 시스템 비활성화

```
[ 곡 제목 / BPM ]          [ 점수 ]
|                                   |
|  레인 1 ─────────────────── [■]  |  ← 판정선
|  레인 2 ────────── [■] ───────  |

```

### 6-2. 결과 화면 (Result Screen)

곡 종료 후 표시되는 화면. 화면 어디든 클릭하면 스토리 선택 화면으로 복귀.

**레이아웃**
```
┌─────────────────────────────────┐
│                                 │
│          [ 등급: S ]            │
│                                 │
│           97.50%                │  ← 정확도, 크게 표시
│                                 │
│  PERFECT  120   GREAT   8       │
│  GOOD       2   MISS    0       │
│                                 │
│        Score: 128,500           │
│                                 │
│   ( 클릭하면 선택 화면으로 )   │
└─────────────────────────────────┘
```

**정확도(Accuracy) 산출식**
```
Accuracy = (PERFECT×1.0 + GREAT×0.8 + GOOD×0.5) / 전체 노트 수 × 100
```

| 표시 요소 | 설명 |
|-----------|------|
| 등급 | S+/S/A/B/C/D — 정확도 % 기반 (등급 기준은 §4 참고) |
| 정확도 | 0.00% ~ 100.00%, 화면 중앙에 크게 표시 |
| 판정 카운트 | PERFECT / GREAT / GOOD / MISS 각 달성 개수 |
| 점수 | 최종 누적 점수 |
| 화면 전환 | 클릭 → 스토리 선택 화면으로 복귀 |

### 메인 메뉴 구성
1. **곡 선택** — 라이브러리 브라우저 (커버 아트, BPM, 난이도 표시)

3. **설정** — 키 바인딩, 오프셋 조정, 스크롤 속도, 배경 밝기
4. **튜토리얼**

### 6-3. 오프셋 보정 (Offset Calibration)

유저의 입력 지연(모니터/오디오 레이턴시 등)을 보정하기 위해 판정선 위치를 픽셀 단위로 좌/우 이동하는 기능.

**판정선 위치 기준**

| 항목 | 값 |
|------|----|
| 기본 위치 | 화면 좌측 15% 지점 |
| 조절 범위 | ±10%p (5% ~ 25%) |
| 조절 단위 | 픽셀 (내부 저장은 화면 비율) |

**이동 방향의 의미**
- **왼쪽 이동** → 노트 이동 거리 단축 → 타이밍 앞당김 (입력이 늦은 유저용)
- **오른쪽 이동** → 노트 이동 거리 연장 → 타이밍 늦춤 (입력이 빠른 유저용)

**설정 방법 (테스트 곡 타격 방식)**
1. 설정 메뉴 → "오프셋 보정" 진입
2. 짧은 테스트 패턴(메트로놈 비트 기반) 재생
3. 유저가 비트에 맞춰 타격
4. 타격 오차 분포를 판정 히트바로 시각화
5. 슬라이더로 판정선 위치 좌/우 조절 → 히트바 기준선 실시간 반영
6. 확인 시 저장

**설정 화면 레이아웃**
```
[ 오프셋 보정 ]

  ◀ ──────────●────────── ▶
  -10%    기본(15%)    +10%

  현재 판정선 위치: 18%

  [테스트 재생]  [초기화]  [확인]

  [판정 히트바 — 오차 분포 시각화]
```

## 8. 스토리 모드 구조

> 세부 내용(에피소드·챕터·스테이지·음악 방향 등)은 별도 문서를 참고:
> **[스토리모드 기획서 → StoryMode_GDD.md](StoryMode_GDD.md)**

### 계층 구조 요약
```
게임
└── 에피소드 (4개): 그의 이야기 / 그녀의 이야기 / 고양이의 이야기 / 가족의 이야기
    └── 챕터 (여러 개 — 각 챕터는 하나의 사건을 단편적으로 표현)
        └── 스테이지 (챕터당 4개 고정 — 기·승·전·결)
            └── 음악 1곡
```

### 스테이지 진행 & 해금 규칙

- **시작 시 EP1 CH1 Stage1만 해금**, 나머지는 잠금
- 스테이지에서 **A 랭크 이상(정확도 ≥ 80%)** 달성 시 다음 스테이지 해금
- 챕터 마지막(Stage4) 달성 시 다음 챕터 Stage1 자동 해금
- 세부 등급 기준 및 저장 구조는 [StoryMode_GDD.md](StoryMode_GDD.md) 참고

### 시각 연출 방향
- 에피소드마다 고유 색감 팔레트 및 배경 톤
- 스테이지 시작 전 짧은 텍스트 컷신으로 이야기 전달
- 노트 색상·이펙트도 스테이지 감정선에 맞춰 변경

## 9. 오디오 생성 파이프라인

### 개요
유저(개발자)의 실제 인생 경험을 기반으로 스토리라인을 구성하고, 이를 AI 음악 생성 서비스에 프롬프트로 전달해 음악을 제작하는 파이프라인.

### 파이프라인 흐름
```
[유저 인생 이야기 전달]
        ↓
[Claude: 스토리라인 & 챕터 구성]
        ↓
[Claude: 각 챕터별 음악 분위기/장르/무드 정의 → 프롬프트 생성]
        ↓
[AI 음악 생성 서비스 (Suno / Udio 등): 프롬프트 기반 음악 생성]
        ↓
[음악 파일(.mp3/.ogg) → 게임에 탑재]
        ↓
[채보 제작 (수동 or 자동 BPM 분석 기반)]
```

### AI 음악 생성 서비스 후보
| 서비스 | 특징 |
|--------|------|
| **Suno** | 가사 포함 가능, 다양한 장르, 고품질 |
| **Udio** | 세밀한 분위기 제어, 스타일 커스터마이징 |

### 채보 연동
- 생성된 음악의 BPM을 분석 (Godot 내장 or 외부 툴)
- BPM 기반으로 채보 데이터(JSON) 수작업 또는 반자동 생성

## 10. 채보 포맷 (Beatmap JSON)

### 파일 구조

```json
{
  "meta": {
    "title": "곡 제목",
    "artist": "아티스트",
    "bpm": 120,
    "audio_file": "res://audio/ep1_ch1_stage1.ogg",
    "duration_ms": 180000,
    "difficulty": "normal",
    "lane_count": 4,
    "preview_start_ms": 30000
  },
  "notes": [
    { "time_ms": 1000, "lane": 2, "type": "tap" },
    { "time_ms": 2000, "lane": 1, "type": "hold", "duration_ms": 800 },
    { "time_ms": 3500, "lane": 3, "type": "slide", "duration_ms": 1200, "required_hits": 6 }
  ]
}
```

### meta 필드

| 필드 | 타입 | 설명 |
|------|------|------|
| title | string | 곡 제목 |
| artist | string | 아티스트명 |
| bpm | number | 기준 BPM |
| audio_file | string | Godot 리소스 경로 (res://) |
| duration_ms | number | 곡 전체 길이 (ms) |
| difficulty | string | `easy` / `normal` / `hard` / `expert` |
| lane_count | number | 레인 수 (4, 6, 8 중 하나) |
| preview_start_ms | number | 곡 선택 화면 미리듣기 시작 위치 (ms) |

### notes 배열 — 공통 필드

| 필드 | 타입 | 설명 |
|------|------|------|
| time_ms | number | 판정선 도달 시각 (곡 시작 기준 ms) |
| lane | number | 레인 번호 (1-based) |
| type | string | `tap` / `hold` / `slide` |

### notes 배열 — 타입별 추가 필드

| type | 추가 필드 | 타입 | 설명 |
|------|-----------|------|------|
| hold | duration_ms | number | 버튼을 누르고 있어야 하는 시간 (ms) |
| slide | duration_ms | number | 슬라이드 전체 지속 시간 (ms) |
| slide | required_hits | number | 성공에 필요한 연타 횟수 |

### 파일 명명 규칙

```
ep{에피소드번호}_ch{챕터번호}_stage{스테이지번호}_{난이도}.json
예) ep1_ch1_stage2_normal.json
```
