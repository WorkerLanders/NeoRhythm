# NeoRhythm — 게임 기획서 초안 (GDD v0.1)

## 개요

Windows 플랫폼 대상의 횡스크롤 리듬 게임. 노트가 우→좌로 이동하며 판정선 도달 시 마우스 클릭 또는 키 입력으로 타격하는 방식. 유저와 가족의 실제 인생 경험을 바탕으로 한 스토리 모드를 핵심으로 하며, AI 음악 생성 서비스를 통해 이야기에 맞는 음악을 제작한다.

| 항목 | 내용 |
|------|------|
| 장르 | 횡스크롤 리듬 액션 |
| 플랫폼 | Windows (PC) |

## 2. 핵심 메카닉

### 2-1. 레인 구성
- 화면을 수평으로 **4레인 고정** (전 난이도 공통, 확장 없음)
- **모든 레인은 단 하나의 입력(마우스 좌클릭)에 대응** — 레인 구분은 시각적 다양성 목적
- 판정선은 화면 **좌측 약 15% 지점**에 수직으로 고정
- **동시에 2개 이상의 레인에 노트가 나오는 경우 없음** (채보 설계 원칙)
- 난이도 차별화는 레인 수가 아닌 **노트 밀도 및 패턴 복잡도**로 구분

### 2-2. 입력 방식
- **마우스 좌클릭 고정** (단일 입력)
- 어느 레인의 노트든 같은 클릭 하나로 처리
- 클릭 타이밍만이 판정에 영향을 미침

### 2-3. 노트 종류 (기본 3종)
| 종류 | 설명 | 입력 |
|------|------|------|
| **탭 노트** | 기본 단타 노트 | 마우스 좌클릭 단타 |
| **홀드 노트** | 길게 누르는 노트 (롱노트) | 마우스 버튼 누름 유지 |
| **슬라이드 노트** | 연속 연타 (드럼롤 느낌) | 연속 클릭 (BPM 기반 필요 횟수 자동 계산) |

### 2-4. 노트 흐름
- 노트 생성 위치: 화면 오른쪽 바깥 (오프스크린)
- 이동 방향: 우 → 좌 (일정 속도, 노래 BPM 기반으로 스크롤 속도 조절 가능)
- 한 시점에 활성 노트는 항상 **1개 레인**에만 존재
- 판정선 통과 시 판정 발생 → 통과하면 Miss 처리

### 2-5. 슬라이드 노트 성공 조건 공식

슬라이드 구간 내 필요 클릭 수는 BPM 기반으로 자동 계산한다:

```
required_clicks = floor((duration_sec × BPM) / 60 × 0.5) + 1
```

- `duration_sec`: 슬라이드 노트 구간 길이 (초)
- `BPM`: 해당 구간의 곡 BPM
- `0.5`: 8분음표 기준 계수 (구현 시 튜닝 포인트)
- 최소 1회 보장 (`+1`)

**예시 (BPM 120, 2초 구간):** `floor(2 × 120 / 60 × 0.5) + 1 = 3회`

구간 내 `required_clicks` 이상 클릭하면 성공, 미달 시 MISS 처리.

## 3. 판정 시스템

| 판정 | 타이밍 오차 | 배율 |
|------|------------|------|
| **PERFECT** | ±20ms | ×1.0 |
| **GREAT** | ±50ms | ×0.8 |
| **GOOD** | ±100ms | ×0.5 |
| **MISS** | 초과 / 미입력 | ×0 (콤보 리셋) |

- 판정 결과는 판정선 위에 텍스트 + 이펙트로 즉시 표시
- 화면 하단에 **판정 히트바** (오차 분포 시각화) 표시 고려

---

## 4. 스코어링 & 콤보

- **기본 점수**: 노트당 1000점 × 판정 배율
- **콤보 보너스**: 콤보 수에 따라 추가 배율 (50콤보: ×1.1 / 100콤보: ×1.2 / 200콤보: ×1.5 등)
- **MAX 콤보**: 곡 내 모든 노트 PERFECT 시 달성
- **등급 시스템**: S+ / S / A / B / C / D (정확도 % 기반)

---

## 5. 체력(HP) 시스템 (선택적)

- HP 바를 화면 일측에 표시
- MISS 시 HP 감소, PERFECT/GREAT 시 소폭 회복
- HP 0 → 곡 실패 (arcade 모드)
- 연습 모드에서는 HP 시스템 비활성화

## 7. 난이도 체계

| 레벨 | 이름 | 특징 |
|------|------|------|
| 1–3 | Easy | 단타 위주, 저밀도, 긴 노트 간격 |
| 4–6 | Normal | 홀드 노트 등장, 노트 간격 증가 |
| 7–9 | Hard | 슬라이드 노트 등장, 연타 패턴 증가 |
| 10–12 | Expert | 고밀도, 복합 패턴 (탭+홀드+슬라이드 혼합), 빠른 BPM |

> 전 난이도 4레인 고정. 난이도 차별화는 노트 밀도·패턴 복잡도로만 구분.

## 8. 스토리 모드 구조

> 세부 내용(에피소드·챕터·스테이지·음악 방향 등)은 별도 문서를 참고:
> **[스토리모드 기획서 → StoryMode_GDD.md](StoryMode_GDD.md)**

### 계층 구조 요약
```
게임
└── 에피소드 (4개): 그의 이야기 / 그녀의 이야기 / 고양이의 이야기 / 가족의 이야기
    └── 챕터 (여러 개 — 각 챕터는 하나의 사건을 단편적으로 표현)
        └── 스테이지 (챕터당 4개 고정 — 기·승·전·결)
            └── 음악 1곡
```

### 시각 연출 방향
- 에피소드마다 고유 색감 팔레트 및 배경 톤
- 스테이지 시작 전 짧은 텍스트 컷신으로 이야기 전달
- 노트 색상·이펙트도 스테이지 감정선에 맞춰 변경

## 9. 오디오 생성 파이프라인

### 개요
유저(개발자)의 실제 인생 경험을 기반으로 스토리라인을 구성하고, 이를 AI 음악 생성 서비스에 프롬프트로 전달해 음악을 제작하는 파이프라인.

### 파이프라인 흐름
```
[유저 인생 이야기 전달]
        ↓
[Claude: 스토리라인 & 챕터 구성]
        ↓
[Claude: 각 챕터별 음악 분위기/장르/무드 정의 → 프롬프트 생성]
        ↓
[AI 음악 생성 서비스 (Suno / Udio 등): 프롬프트 기반 음악 생성]
        ↓
[음악 파일(.mp3/.ogg) → 게임에 탑재]
        ↓
[채보 제작 (수동 or 자동 BPM 분석 기반)]
```

### AI 음악 생성 서비스 후보
| 서비스 | 특징 |
|--------|------|
| **Suno** | 가사 포함 가능, 다양한 장르, 고품질 |
| **Udio** | 세밀한 분위기 제어, 스타일 커스터마이징 |

### 채보 연동
- 생성된 음악의 BPM을 분석 (Godot 내장 or 외부 툴)
- BPM 기반으로 채보 데이터(JSON) 수작업 또는 반자동 생성

## 10. 채보 JSON 스키마

### 설계 원칙
- 시간 표현: **ms 절대값** 방식 (Godot `AudioStreamPlayer.get_playback_position() × 1000`과 직접 비교)
- 레인 배정: 채보에 포함하지 않고 **런타임 랜덤 배정** (직전 노트와 다른 레인 보장)

### 스키마 정의

```json
{
  "meta": {
    "title": "탄생, 그리고 소중함 - Stage 1",
    "artist": "AI Generated (Suno)",
    "bpm": 90,
    "duration": 142.5,
    "audio_file": "ep1_ch1_s1.ogg",
    "charter": "dev",
    "version": "1.0"
  },
  "notes": [
    {
      "id": 0,
      "type": "tap",
      "time": 1333
    },
    {
      "id": 1,
      "type": "hold",
      "time": 3000,
      "duration": 1500
    },
    {
      "id": 2,
      "type": "slide",
      "time": 6000,
      "duration": 2000
    }
  ]
}
```

### 필드 설명

**`meta`**

| 필드 | 타입 | 설명 |
|------|------|------|
| `title` | string | 곡/스테이지 제목 |
| `bpm` | number | 곡 BPM (슬라이드 필요 클릭 수 계산에 사용) |
| `duration` | number | 곡 전체 길이 (초) |
| `audio_file` | string | 오디오 파일명 |
| `charter` | string | 채보 제작자 |
| `version` | string | 채보 버전 |

**`notes` 배열**

| 필드 | 타입 | 적용 노트 | 설명 |
|------|------|-----------|------|
| `id` | int | 전체 | 노트 고유 번호 (순서 보장) |
| `type` | string | 전체 | `"tap"` / `"hold"` / `"slide"` |
| `time` | int | 전체 | 판정선 도달 목표 시각 (ms) |
| `duration` | int | hold, slide | 구간 길이 (ms) — tap은 생략 |

### 레인 랜덤 배정 (Godot 구현 참고)

```gdscript
var last_lane = 0
func get_random_lane() -> int:
    var lane = randi() % 4 + 1
    while lane == last_lane:
        lane = randi() % 4 + 1
    last_lane = lane
    return lane
```

노트 스폰 시 `get_random_lane()`을 호출하여 레인을 결정한다.
