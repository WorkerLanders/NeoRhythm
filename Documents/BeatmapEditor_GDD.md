# NeoRhythm 채보 에디터 기획서

> 참조 문서: [GDD_v0.1.md](GDD_v0.1.md) — Section 13 채보 JSON 스키마

---

## 1. 개요

### 목적

GDD에 정의된 채보 JSON 형식을 시각적으로 제작하기 위한 도구.  
오디오 파일을 로드하고 타임라인 위에 탭·홀드 노트를 배치한 뒤 JSON으로 내보낸다.

### 플랫폼

**단일 HTML 파일 (`tools/beatmap_editor.html`)**

- 서버 불필요, 브라우저에서 바로 실행
- PC 없이 웹 브라우저만으로 접근 가능 (게임 개발 환경과 동일한 Web 우선 방침)
- 오디오 처리: Web Audio API
- 화면 렌더링: HTML5 Canvas
- 파일 입출력: File API

### 출력 형식

GDD_v0.1.md Section 13 스키마 호환 JSON.  
파일명 규칙: `ep{N}_ch{N}_stage{N}_{difficulty}.json`

## 3. 노트 배치 인터랙션

| 동작 | 결과 |
|------|------|
| 노트 트랙 좌클릭 | **탭 노트** 배치 — 클릭한 위치의 ms → `time` |
| 노트 트랙 좌클릭 후 드래그 | **홀드 노트** 배치 — 드래그 시작 ms → `time`, 드래그 길이 ms → `duration` |
| 노트 위 우클릭 | 노트 삭제 |
| 노트 좌클릭 후 드래그 | 노트 이동 (`time` 변경) |
| 홀드 노트 끝점 드래그 | 홀드 길이 조절 (`duration` 변경) |

> 레인은 채보에 포함하지 않으므로 (GDD 원칙: 런타임 랜덤 배정) 에디터에서도 레인 선택이 없다.

## 4. BPM 그리드 & 스냅

- BPM 입력 시 타임라인에 박자 눈금 자동 표시
- **스냅 단위** 선택 드롭다운: `1박` / `1/2박` / `1/4박` / `1/8박` / `스냅 없음`
- 노트 배치·이동 시 가장 가까운 스냅 눈금으로 자동 정렬

**스냅 간격 공식:**
```
스냅 간격(ms) = 60000 ÷ BPM ÷ 박자분모
```

| BPM | 스냅 단위 | 간격 |
|-----|----------|------|
| 90  | 1박      | 666.7ms |
| 90  | 1/4박    | 166.7ms |
| 120 | 1/4박    | 125ms |

## 5. 타임라인 조작

| 입력 | 동작 |
|------|------|
| 마우스 휠 | 수평 스크롤 |
| Ctrl + 마우스 휠 | 줌 인/아웃 (시간축 확대·축소) |
| 스페이스바 | 재생/정지 토글 |
| 타임라인 빈 공간 클릭 | 재생 헤드 이동 |

- **줌 범위**: 1px = 1ms (최대 줌) ~ 1px = 100ms (최소 줌)
- **재생 중 자동 스크롤**: 재생 헤드가 화면 중앙에 고정되도록 타임라인 자동 이동

## 6. 메타 정보 입력

| 항목 | 입력 방식 | 출력 JSON 필드 | 비고 |
|------|----------|---------------|------|
| 제목 | 텍스트 | `meta.title` | |
| 아티스트 | 텍스트 | `meta.artist` | |
| 난이도 | 드롭다운 | `meta.difficulty` | `easy` / `normal` / `hard` / `expert` |
| BPM | 정수 입력 | `meta.bpm` | 그리드 스냅에도 실시간 연동 |
| 오디오 파일명 | 오디오 로드 시 자동 입력 | `meta.audio_file` | |
| 채보 제작자 | 텍스트 | `meta.charter` | |
| 곡 길이 | **자동 산출** | `meta.duration` | Web Audio API `AudioBuffer.duration` |
| 버전 | 고정값 | `meta.version` | `"1.0"` 고정 |

## 7. JSON 출력 명세

JSON 저장 버튼 클릭 시 브라우저 다운로드로 파일 생성.

### 파일명

저장 전 입력 대화상자에서 파일명 확인:
```
ep{N}_ch{N}_stage{N}_{difficulty}.json
예) ep1_ch1_stage1_easy.json
```

### 자동 처리 규칙

| 항목 | 처리 |
|------|------|
| `notes` 정렬 | `time` 오름차순 자동 정렬 |
| `id` 부여 | 정렬 후 0부터 순서대로 자동 부여 |
| `duration` 필드 | 홀드 노트만 포함 — 탭 노트는 필드 생략 |

### 임시 저장

- 편집 중 **LocalStorage에 자동 저장** (30초 간격)
- 브라우저 새로고침 또는 재접속 시 마지막 작업 자동 복구
- "새 채보" 버튼으로 초기화

### 출력 예시

```json
{
  "meta": {
    "title": "탄생, 그리고 소중함 - Stage 1",
    "artist": "AI Generated (Suno)",
    "difficulty": "easy",
    "bpm": 90,
    "duration": 142.5,
    "audio_file": "ep1_ch1_s1.ogg",
    "charter": "dev",
    "version": "1.0"
  },
  "notes": [
    { "id": 0, "type": "tap",  "time": 1333 },
    { "id": 1, "type": "tap",  "time": 2667 },
    { "id": 2, "type": "hold", "time": 4000, "duration": 1500 },
    { "id": 3, "type": "tap",  "time": 6000 }
  ]
}
```

## 8. 기술 스택

| 항목 | 선택 |
|------|------|
| 실행 환경 | 브라우저 (단일 HTML 파일, 서버 불필요) |
| 오디오 처리 | Web Audio API (`AudioContext`, `decodeAudioData`) |
| 파형 렌더링 | HTML5 Canvas (오디오 PCM 데이터 직접 그리기) |
| 타임라인 렌더링 | HTML5 Canvas |
| JSON 출력 | `Blob` + `URL.createObjectURL` 다운로드 |
| 임시 저장 | `localStorage` |
| 외부 의존성 | **없음** (순수 HTML + Vanilla JS) |
